## 协作规则

1. **文档优先**：玩法/数值变更先改 `docs/`，再改 `src/`
2. **先列计划再动手**：需求有歧义或复杂度较高时，先出一份计划跟阿燃确认具体细节，再开始写代码
3. **分步推进**：功能不要一次做太多，拆成小步骤逐步确认，每步做完再继续
4. **提交规范**：每次功能完成后审查并 Git 提交
5. **推送**：审查 → Git commit → push 到 GitHub master
